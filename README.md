# Duchenne-Muscular-Dystrophy-long-read-transcriptomics

This repository contains the computational pipelines and bioinformatic workflows developed for my **Final Degree Project (TFG) in Biomedical Sciences** at the **Instituto Químico de Sarria (IQS)**, Barcelona. The project focuses on performing a transcriptomic analysis of Duchenne Muscular Dystrophy mouse models using long-read Nanopore sequencing to identify candidate biomarkers for personalised genetic therapies.

## EXPERIMENTAL OVERVIEW
* **Animal model:** Three-month-old male mice (Wild-Type C57BL/6 vs. D2-Mdx dystrophic models).
* **Target tissue:** Cardiac muscle (whole heart RNA extraction).
* **Sequencing platform:** Oxford Nanopore Technologies (ONT) MinION.
* **Chemistry and library preparation:** R10.4.1 flow cells (FLO-MIN114) and cDNA-PCR Sequencing Kit with Barcoding (SQK-PCB114.24).

### BIOINFORMATIC FRAMEWORK

### Initial data processing and quality control (common pre-processing)
Before executing any of the specific exploratory or definitive pipelines, the raw current signals generated by the MinION device undergo a standardized pre-processing workflow:
1. **Basecalling:** Translation of raw electrical signals into nucleotide sequences using **Dorado v1.1.1**.
2. **Demultiplexing:** Sorting and separating the basecalled reads into individual sample files based on their unique barcode sequences via **Dorado demux**.
3. **Formatting:** Converting the demultiplexed data from BAM format into standard compressed FASTQ files using **Samtools v1.22.1**.
4. **Quality control:** Generating initial throughput metrics, read length distributions (N50), and mean Phred quality scores (Q) across all processed replicates using **NanoPlot v2.26-r1175**.

During the exploratory phase of the project, five alternative bioinformatic processing workflows were evaluated and compared to establish the most robust processing method:

### Pipeline 1A: Index-based quantification with Limma
* **Quantification:** Using Salmon in quasi-mapping mode.
* **Statistical modelling:** Differential expression analysis conducted in RStudio using the Limma package following a direct logarithmic transformation (log_2(TPM + 1)).

### Pipeline 1B: Index-based quantification with DESeq2
* **Quantification:** Using Salmon in quasi-mapping mode.
* **Statistical modelling:** Negative binomial modeling on raw transcript counts in RStudio using the DESeq2 package, combined with a variance-stabilizing transformation (vst).

### Pipeline 1C: Index-based quantification with Limma-Voom
* **Quantification:** Using Salmon in quasi-mapping mode.
* **Statistical modelling:** Filtering of low-expressed transcripts and Trimmed Mean of M-values (TMM) normalization, followed by mean-variance precision weighting via the edgeR-Voom and Limma workflows.

### Pipeline 2: Alignment-based transcript quantification
* **Mapping:** Long reads aligned directly to the reference transcriptome using Minimap2 in a splice-aware manner.
* **Quantification and modelling:** Isoform abundance estimation using Salmon in alignment-based mode, with downstream differential transcript expression analysis via DESeq2.

### Pipeline 3: Genome alignment for gene-level quantification
* **Mapping:** Long reads aligned directly to the reference genome using Minimap2.
* **Quantification and modelling:** Gene-level feature counting using the `featureCounts` function to collapse reads into overall gene blocks, with subsequent differential gene expression analysis via DESeq2.

Following a comparative assessment of robustness, data distribution, and statistical power during the testing phase, **Pipeline 3** (Genome alignment for total gene-level changes) and **Pipeline 1B** (Index-based quantification for high-resolution individual transcript analysis) were selected as the complementary definitive workflows to process the final high-throughput sequencing datasets.

### REPOSITORY STRUCTURE AND USAGE

1. Ensure your conda environments are set up with the required packages and libraries.
2. Adapt the  storage directory paths (`<path>`) before executing script sequences.
3. For downstream data visualization (Volcano plots, interactive sample PCA, and hierarchical clustering heatmaps), scripts should be executed natively within RStudio environment using `Bioconductor v3.23` repositories.
